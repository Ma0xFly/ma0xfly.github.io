#车联网

来源路径：车联网/硬件.md

嵌入式系统芯片类型

1. MCU（Micro Control Unit，微控制器）低配微型计算机

  • 定义：
    将CPU、RAM、ROM（或Flash）、定时计数器、I/O接口（如UART、SPI、ADC）等计算机核心组件集成在单一芯片上，形成完整的微型计算机系统 。 资源有限，适合执行逻辑简单的特定任务。
  • 特点：
    • 高度集成：无需外接RAM/ROM即可运行代码，可直接通过电阻、电容等简单外围器件构建最小系统 。
    • 低功耗与低成本：采用精简指令集（RISC），主频较低（通常≤300MHz），适合嵌入式控制场景 。
    • 实时控制能力：专为控制任务设计，擅长处理多源数据（如传感器输入） 。
  • 典型应用：
    家电控制（如洗衣机，空调）、工业传感器、汽车仪表等低复杂度场景 。
  • 代表芯片：
    STM32F103（ARM Cortex-M3内核，72MHz主频）、8051、AVR系列 。

与高性能cpu对比

  • CPU（中央处理器）：
    • 性能强悍，适用于高性能计算场景（如电脑、服务器等）。
    • 运行依赖众多外围器件（如内存、显卡、主板芯片组等）。
    • 主流厂商为英特尔、AMD等。
    • 特点：价格较高，工作频率与功率大，存储容量大（通常以GB为单位），多为64位架构。
    • 常见通讯接口：USB等。
  • 单片机（MCU）：
    • 为低性能需求场景设计（如家电、嵌入式设备等），是“迷你版计算机”。
    • 内部集成多种组件（如CPU核心、内存、I/O接口等），无需大量外围器件即可工作。
    • 生产厂商众多，选择范围广。
    • 特点：价格低廉，工作频率与功率低，存储容量小（通常以KB为单位，与CPU内存相差可达百万倍），以8位和32位架构为主。
    • 常见通讯接口：I²C等。

2. MPU（Micro Processor Unit，微处理器）

  • 定义：
    功能强大的CPU，通常不集成RAM/ROM等外围器件，需外接内存才能构成完整系统 。
  • 特点：
    • 高性能：支持复杂计算（如64位运算），主频可达GHz级，可运行多任务操作系统（如Linux）。
    • 通用性强：未针对特定场景优化，需搭配复杂外设（如以太网、Wi-Fi模块）。
    • 接口丰富：提供高速通信接口（USB、PCIe等），扩展性强 。
  • 与MCU区别：
    MCU集成外围器件，MPU需外部扩展；MCU适合单一控制回路，MPU支持多进程/线程 。
  • 典型应用：
    个人电脑、高端工作站、服务器 。
  • 代表芯片：
    Intel x86系列、ARM Cortex-A系列（如全志A20）。

3. SoC（System on Chip，片上系统）

  • 定义：
    在单一芯片上集成MCU的完整性与MPU的高性能，同时包含GPU、DSP、无线模块等专用组件，形成"系统级"解决方案 。 可以理解为把主板上的部分功能嵌入到cup芯片上。
  • 特点：
    • 超强集成度：融合CPU+GPU+DSP+RAM/ROM+通信模块（如Wi-Fi/蓝牙），大幅减少外围电路 。
    • 支持操作系统：可运行Linux、Android等复杂系统 。
    • 能效比高：通过硬件加速模块优化特定任务（如AI计算），降低功耗 。
  • 与MCU/MPU关系：
    SoC = MCU（基础控制） + MPU（高性能处理） + 专用加速单元 。
  • 典型应用：
    智能手机（如苹果A系列芯片）、平板电脑、物联网网关 。
  • 代表芯片：
    苹果A系列、高通骁龙、华为麒麟 。

4. 三者的核心区别总结

特性 MCU MPU SoC
集成度 芯片级（CPU+RAM+ROM+I/O） 仅CPU核心 系统级（CPU+GPU+DSP+外设）
是否需要外接存储器 否 是 部分集成，可扩展
主频 低（MHz级） 高（GHz级） 高（GHz级）
操作系统支持 无或RTOS Linux/Windows等 完整操作系统
典型场景 简单控制（家电、传感器） 计算机/工作站 智能终端（手机、平板）
成本 低（10） 高 中到高

注：随着技术发展，MCU与MPU界限模糊（如部分MCU支持Linux），而SoC已成为智能设备的主流方案 。

硬件串口及调试法

jtag

基础知识

是什么

jtag既是一套边界扫描（Boundary-Scan）测试规范（IEEE-1149.1），又是今天几乎所有 SoC、MCU、FPGA 用来调试、烧录、测试的“万能后门”。一句话：JTAG = 芯片自带的“USB调试口 + 烧录器 + 逻辑分析仪”三合一，只是走专用 4-5 根线。“只要 4 根线、1 个状态机、32 位 IDCODE，就能把整颗 SoC 当 RAM 随意读/写/改寄存器、单步跑、烧 Flash、测引脚。”

1 物理接口

• 最简 4 线：TCK（时钟）、TMS（状态机）、TDI（数据入）、TDO（数据出）
• 常见 5 线：再加 TRST（低电平复位，可选）
• 排列：20/14/10/6/2×5 1.27 mm JTAG 头，或干脆就是 PCB 上的 5 个测试点。
• 电平：1.2 V-3.3 V，得先匹配目标板。

2 内部结构

• TAP（Test Access Port）控制器
‑ 16 个状态的有限状态机，靠 TMS 在 TCK 上升沿改变状态。
• 指令寄存器 IR + 数据寄存器 DR
‑ IR 决定接下来 DR 的功能：IDCODE、BYPASS、EXTEST、DEBUG 等。
• 边界扫描单元
‑ 每个 IO 引脚旁边都有一个 1 bit 的移位寄存器，能“串起来”控制/观测芯片管脚。

3 三条最常用的“指令”

指令 作用 典型场景
IDCODE 读出芯片 32-bit ID 确认芯片型号、验证 JTAG 通路
BYPASS 把芯片 TDI→TDO 直连 1 bit 链上多芯片时缩短移位长度
DEBUG 进入 CPU 的调试模块 设置断点、单步、读写内存/寄存器

4 调试流程（以 ARM Cortex-M 为例）

  1. 连接 J-Link/OpenOCD → 发送 JTAG reset (5 个 TCK + TMS=1)。
  2. 扫描链：Shift-IR 写 IDCODE，Shift-DR 读回 0x4BA00477。
  3. 进入 DEBUG 指令 → 通过 DAP（Debug Access Port）
    ‑ 读写 AHB 总线 → 下载程序、设断点、读 RAM。
  4. 烧录 Flash：把 flashloader 下载到 SRAM，再通过 flashloader 写片内 Flash。

jtagulator探测接口

1. 硬件连接

  • 接地连接：
    万用表蜂鸣档找出目标板 GND，将JTAGulator的GND引脚与目标设备的接地（GND）连接，确保共地 。
  • 通道连接：
    将目标设备的可疑测试点（如PCB上的焊盘、过孔）连接到JTAGulator的CH0至CH23通道（最多24通道）。
  • 电压设置：
    用万用表测量目标设备电压，通过V命令设置JTAGulator的VADJ引脚电压（1.2V–3.3V），避免损坏设备

2. 串口配置

  • 连接电脑：
    通过USB转串口模块连接JTAGulator与电脑，使用串口终端（xshell）。
  • 参数设置：
    波特率 115200，数据位 8，停止位 1，无校验位 。

3. 扫描JTAG引脚

  • 进入JTAG模式：
    串口输入J进入JTAG操作菜单 。
  • IDCODE扫描（推荐）：
    输入I启动快速扫描。JTAGulator会遍历所有引脚组合，检测是否返回有效IDCODE（非全0/全1的32位码）。若成功，输出类似：
    Found! TDO: CH1, TCK: CH4, TMS: CH3（引脚编号可能不同）。
    • 原理：JTAG标准要求支持IDCODE寄存器，扫描通过TDO返回设备ID 。
  • BYPASS扫描（备用）：
    若IDCODE扫描失败（目标禁用IDCODE），输入B启动BYPASS模式扫描。该模式检测TDI输入与TDO输出的延迟匹配（输入数据经1个时钟周期后输出）。
    • 补充TDI：扫描后若TDI未识别，输入T并指定已知引脚（如CH1,CH4,CH3,x），JTAGulator自动检测TDI 。

电表使用

找GND

  1. 断电 → 蜂鸣档 → 黑笔接金属外壳/电瓶负极。
  2. 红笔挨个碰待测点，“滴”一声 < 0.5 Ω → GND。
