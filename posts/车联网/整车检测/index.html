wifi

安全模式

wep 是 Wired Equivalent Privacy(有线等效保密)的英文缩写,目前常见的是 64 位 WEP 加密和 128 位 WEP加密。它是一种最老也是最不安全的加密方式,虽然无线路由器都支持这种加密方式,但是安全性能太差,非常容易被破解。

wpa 是 WEP 加密的改进版,包含两种方式:预共享密钥和 Radius 密钥(远程用户拨号认证系统)。其中预共享密钥(pre-share key 缩写为 PSK)有两种密码方式:TKIP 和 AES,而 RADIUS 密钥利用 RADIUS 服务器认证并可以动态选择 TKIP、AES、WEP 方式。相比 TKIP,AES 具有更好的安全系数,不容易被破解。
wpa2 即 WPA 加密的升级版。WPA2 同样也分为 TKIP 和 AES 两种方式,不容易被破解。

现在在生活中,很少能碰到 wep 加密的网线网络了,几乎都是 wpa 类型的,或许还能看见几个开启 wps 功能的无线网络。

连接模式

谁开热点就是ap。连接方是sta

车机（Android Automotive / Linux 车机 / QNX Hypervisor）把 Wi-Fi 当 4 种角色用：

  1. STA 模式 – 车机自己上网（导航、OTA、影音）。
  2. AP 模式 – 车机变成 车载热点，给手机/平板上网。
  3. STA + AP 并发 – 一边连蜂窝/外来热点，一边给乘客开热点。
  4. P2P（Wi-Fi Direct） – 投屏、CarPlay/Android Auto Wireless。

蜂窝网络

对比维度 蜂窝网络（如 4G/5G） WiFi（如 WiFi 5/6）
覆盖范围 广域覆盖：基站信号可覆盖城市、郊区、公路等，理论上只要有基站的地方就能连接（如户外、偏远地区）。 短距离覆盖：信号受障碍物（墙、金属）影响大，覆盖半径通常 10-100 米（如家庭、办公室、咖啡馆内）。
连接前提 1. 设备需支持对应网络制式（如 5G NR）； 2. 需插入运营商 SIM 卡（或开通 eSIM）； 3. 需在运营商信号覆盖范围内。 1. 设备需支持 WiFi 协议； 2. 需知道目标 WiFi 的名称（SSID）和密码（公开 WiFi 无需密码）； 3. 需在路由器信号覆盖范围内。
数据来源 运营商分配的 “移动数据流量”（需购买流量套餐，超出后可能限速 / 收费）。 依赖路由器后端的 “有线宽带”（如家庭宽带）或 “蜂窝网络共享”（如手机开热点时，用的是手机自身的移动流量）。
速度与稳定性 速度：4G 理论峰值 100Mbps-1Gbps，5G 理论峰值 10-20Gbps，但实际速度受基站负载、信号强度影响（如高峰时段可能卡顿）； 稳定性：户外移动时（如坐车）可能因切换基站导致短暂断连。 速度：WiFi 5（802.11ac）理论峰值 3.5Gbps，WiFi 6（802.11ax）理论峰值 9.6Gbps，实际速度接近宽带上限（如 100M 宽带≈10MB/s）； 稳定性：固定场景（如家中）信号稳定，无切换基站问题，但多设备连接可能分摊带宽。
使用成本 需支付运营商 “流量套餐费”，超出套餐后按流量计费（成本较高，尤其大流量场景）。 主要成本是 “有线宽带月租”，连接 WiFi 本身免费（无额外流量费），适合长期大流量使用（成本低）。
设备兼容性 需设备支持移动网络模块（如手机、平板、4G/5G 路由器），普通笔记本、智能电视通常不自带（需外接网卡）。 几乎所有智能设备都支持（手机、电脑、电视、智能家居、手环等），兼容性极强。
安全性 依赖运营商加密技术（如 4G 的 EPS 加密、5G 的 256 位加密），传输过程安全性较高，但公共场合的 “伪基站” 可能存在诈骗风险。 安全性依赖 WiFi 设置： - 家用 WiFi 建议用 “WPA3” 加密（比 WPA2 更安全）； - 公共 WiFi（如商场、车站）可能存在 “钓鱼 WiFi”（窃取数据），需谨慎连接。
典型使用场景 户外移动场景：如手机导航、乘车时刷视频、偏远地区无宽带时上网； 无 WiFi 覆盖的场景：如野外作业、长途旅行。 固定场景：如家中追剧、电脑办公、智能家居联网； 公共室内场景：如咖啡馆、图书馆、公司内上网。

nmap

Nmap（Network Mapper）是一款开源网络扫描工具，用于网络发现和安全审计。其主要功能包括：

  • 探测网络主机存活状态
  • 扫描开放端口及运行的服务
  • 识别操作系统类型和版本
  • 检测防火墙配置
  • 支持脚本扩展（NSE）进行漏洞检测

参数

  • 主机发现：扫描网段内存活主机
nmap -sn 192.168.1.0/24    # 仅探测存活主机，不扫描端口[[8, 10]]
#-sn：Ping 扫描（不做端口扫描，只检测主机是否在线）。
#192.168.1.0/24：扫整个网段，从 192.168.1.1 到 192.168.1.254。
  • 扫描：
-p- ：指定端口范围
-p :指定端口

-sT :tcp扫描，完整完成三次握手，耗时，易被目标主机日志记录
-sS ：tcp扫描，syn扫描收到sys ack就rst会话，省时隐蔽，不发rst
会导致目标主机重传信息浪费资源
-sn ：Ping 扫描，可以跟ip范围
-sU :udp扫描


- TCP SYN半开扫描（快速且隐蔽）：  

nmap -sS 目标IP          # 默认扫描1000个常用端口[[3, 6, 18]]

- 指定端口范围：  

nmap -p 80,443,1-1000 目标IP   # 扫描指定端口[[6, 10]]
  • 服务与系统识别：
nmap -sV -O 目标IP         # 检测服务版本和操作系统[[3, 5, 8]]
  • 全面扫描：
nmap -A 目标IP             # 启用OS检测、版本检测、脚本扫描等[[8, 18]]
  • 脚本引擎（NSE）：
nmap --script=vuln 目标IP  # 使用漏洞检测脚本[[3, 5, 18]]

高级选项：

  • -T4：加速扫描（共0-5级）
  • -Pn：跳过主机发现，直接扫描
  • -f：分片数据包以绕过防火墙
  • -D RND:5：使用诱饵IP掩盖真实扫描源

fscan

Fscan 是一款专注于内网渗透的自动化扫描工具，用Go语言编写。其特点包括：

  • 快速探测内网存活主机和常见端口
  • 集成漏洞检测（如未授权访问、弱口令爆破）
  • 支持一键扫描B段/C段网络
    常与Nmap配合使用：Fscan用于快速定位脆弱点，Nmap用于深度扫描。

使用方法

  • 基础扫描：
fscan -h 192.168.1.1       # 扫描单个IP
  • 网段扫描：
fscan -h 192.168.1.1/24    # 扫描C段所有IP
  • 指定漏洞检测：
fscan -h 目标IP -p 21,445 -vuln  # 检测FTP/SMB漏洞

工具对比与协作场景

  • nmap：功能最全、脚本最多，但得自己“拼乐高”。只扫端口
  • fscan：把常见漏洞+弱口令打包好，**一键“内网梭哈”**自动验证漏洞

socat端口转发

socat端口转发是一种通过socat工具实现的网络流量转发机制，它可以在两个端口之间建立虚拟通道，实现数据的传输。

示例 本地端口转发到远程服务器

场景：
你有一台远程服务器（IP：192.168.1.100），它运行了一个 Web 服务在 80 端口，但你无法直接访问它。你有一台中间服务器（IP：192.168.1.200），你可以通过它访问。

目标：
你希望在本地监听一个端口（比如 8080），然后将所有请求转发到远程服务器的 80 端口。

命令（在中间服务器上运行）：

socat TCP-LISTEN:8080,fork TCP:192.168.1.100:80

解释：
TCP-LISTEN:8080：在本地监听 8080 端口。
fork：为每个连接启动一个新进程，支持并发。
TCP:192.168.1.100:80：将请求转发到远程服务器的 80 端口。
在本地浏览器中访问 http://localhost:8080，就会连接到远程服务器的 80 端口。

hping3（ddos）

hping3 是一款开源的命令行网络数据包构造与发送工具，常被称为“增强版 ping”。它工作在 TCP/IP 协议栈层面，可自定义并发送基于 ICMP、TCP、UDP、RAW-IP 等多种协议的探测或攻击数据包

洪水攻击

  • -S：SYN 标志
  • --flood：全速发包

ACK Flood

sudo hping3 -A --flood -p 80 192.168.240.10

SYN Flood （按 Ctrl-C 停掉上一条后再跑）

sudo hping3 -S --flood -p 80 192.168.240.10

ICMP Flood

sudo hping3 -1 --flood 192.168.240.10

畸形报文

TCP校验和错误（-b）

sudo hping3 -c 1 -S -p 80 -b 172.20.28.2

UDP校验和错误

sudo hping3 -2 -c 1 -p 53 -b 8.8.8.8

ICMP校验和错误

sudo hping3 -1 -c 1 -C 8 -K 0 -b 172.20.28.2

TCP同时设置SYN+FIN（协议禁止的组合）

sudo hping3 -c 1 -S -F -p 80 172.20.28.2

BSSID(Basic Service SetIdentifier): AP 的 MAC 地址。

#Data:当前数据传输量。

#/s:过去 10 秒钟内每秒捕获数据分组的数量。

CH(Channel): AP 所在的频道。

MB: AP 的最大传输速度。MB=11 => 802.11b,MB=22 => 802.11b+, MB>22 => 802.11g。后面带.的表
示短封包标头,处理速度快,更利于破解。

ENC(Encryption): 使用的加密算法体系。

CIPHER: 检测到的加密算法。

AUTH(Authority): 认证方式。

ESSID(The Extended Service Set Identifier): AP 的名称。
