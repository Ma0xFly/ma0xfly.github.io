Euler Finance 漏洞复现工作区

这个仓库是面向审计与复现的 Euler Finance 漏洞脚手架，优先服务“先读后写”的分析流程。现在只维护一套源码树：协议源码全部放在 src/euler/，保持接近上游的可编译布局；审计直观性则通过根 README 和协议 README 来补。

根 README 现在兼任事故主页和地址簿，避免在分析初期来回跳转多个上下文文件。

范围说明

这个仓库当前只聚焦 Euler Finance 在以太坊主网上的 DAI 攻击链。 2023 年 3 月 13 日当天存在多笔攻击交易；这里以 Euler Labs 复盘时间线中公开的第一笔攻击交易为锚点，并使用它前一个区块 16817995 作为推荐 fork 点。

快速开始

  1. 参考 .env.example 填写 .env。
  2. 执行 forge build 做一次仅编译的冒烟检查。
  3. 执行 forge test --match-test testExploit -vvv，确认骨架在不 fork 的情况下也能通过。
  4. 执行 source .env && forge test --match-test testExploit -vvv --rpc-url $TENDERLY_RPC_URL --fork-block-number 16817995，运行带 fork 的 PoC 骨架。
  5. 执行 slither . --exclude-dependencies，对当前唯一源码树做静态扫描。

如果没有配置 TENDERLY_RPC_URL，测试会主动跳过依赖 fork 的步骤，只承担编译和结构校验的职责。

攻击路径摘要

  1. 从 Aave V2 借出 30,000,000 DAI 闪电贷。
  2. 存入 Euler，并通过自借把头寸杠杆拉高。
  3. 调用 donateToReserves() 销毁抵押物，但不减少债务。
  4. 主动把账户推入不健康状态。
  5. 通过自清算吃到清算折价。
  6. 归还闪电贷，保留价差利润。

时间线

  1. 2023-03-13 08:50:23 UTC：Euler Finance Exploiter 1 部署 Euler Exploit Contract 2。
  2. 2023-03-13 08:50:35 UTC：Euler Finance Exploiter 3 部署 Euler Exploit Contract 1。
  3. 2023-03-13 08:50:59 UTC：第一笔参考攻击交易执行，从 Aave V2 借出 30,000,000 DAI，并从 Euler 的 DAI 市场抽走约 $8.8M 价值。
