Sentiment 基于 Balancer 只读重入攻击复现

事故概览

  • 协议：Sentiment
  • 链上攻击时间：2023-04-04 17:50:29 UTC
  • 攻击区块：Arbitrum 77026913
  • 推荐 fork 区块：77026912
  • 日期说明：不少公开文章按本地时区写成 2023-04-05，本仓库统一以链上 UTC 时间为准
  • 核心问题：Sentiment 在 borrow() 中先放款、后做健康度检查；而其 Balancer LP 预言机在 Vault.exitPool() 的只读重入窗口里读取到了尚未更新的池子余额
  • 公开损失规模：约 ~1M USD
  • 攻击交易：0xa9ff2b587e2741575daf893864710a5cbb44bb64ccdc487a100fa20741e0f74d

范围说明

这个仓库当前是“复现脚手架”，不是已经补完的最终 PoC。

已完成部分：

  • 事故上下文、时间线与地址簿已经固化
  • src/sentiment/ 下已经整理出一套可编译的 Sentiment 源码树
  • test/ 下已经拆出最小接口、地址常量和 PoC 骨架
  • 关键文档已切到中文

仍需你本地补充的环境项：

  • 一条可用的 Arbitrum archive RPC，建议直接使用 Tenderly Virtual TestNet，并从区块 77026912 fork
  • 对 0x62c5...6403 在攻击区块对应实现地址的最终确认

快速开始

  1. 在 Tenderly 创建新的 Arbitrum One Virtual TestNet，并从区块 77026912 fork。
  2. 复制 .env.example 为 .env，填入 RPC 和 API Key。
  3. 编译项目：
cd /home/myx/漏洞复现/sentiment-poc
forge build
  1. 运行当前脚手架测试：
source .env
forge test --match-test testExploit -vvv --rpc-url $TENDERLY_RPC_URL --fork-block-number 77026912

攻击路径摘要

  1. 攻击者先从 Aave V3 闪电贷 WBTC、WETH、USDC。
  2. 通过 AccountManager 打开 Sentiment 账户并注入初始抵押品。
  3. 与目标 Balancer weighted pool 交互，建立会被 Sentiment LP 预言机引用的 BPT 头寸。
  4. 退出 Balancer 池子时，WETH 转账触发攻击合约 fallback()，而此时 Balancer 还没完成余额更新。
  5. 攻击者在这个重入窗口里继续向 Sentiment 借款，WeightedBalancerLPOracle.getPrice() 读取到被高估的 BPT 价格。
  6. 借出的资产再经过 Aave / Curve 等路径中转，归还闪电贷后留下利润。

源码阅读指南

先看哪些文件

  1. src/sentiment/modules/WeightedBalancerLPOracle/src/balancer/WeightedBalancerLPOracle.sol
  2. src/sentiment/modules/AccountManager/src/core/AccountManager.sol
  3. src/sentiment/modules/AccountManager/src/interface/core/IRiskEngine.sol
  4. test/helpers/SentimentAddresses.sol
  5. test/interfaces/SentimentInterfaces.sol
  6. test/poc.t.sol

为什么先看

  • WeightedBalancerLPOracle.getPrice() 是本次事故最关键的旧状态读取点。它在 Balancer exitPool() 尚未完成余额结算时，通过 vault.getPoolTokens(...) 读到过期余额。
  • AccountManager.borrow() 是 Sentiment 一侧把旧价格转成真实资金流出的边界。它会先出借，再检查 riskEngine.isAccountHealthy(account)。
  • IRiskEngine 能帮助继续定位健康度检查如何依赖预言机价格。
  • test/ 侧文件把真实地址、最小接口和攻击骨架集中到一处，调试时不需要在源码树和测试树之间来回跳转。

漏洞相关入口

  • WeightedBalancerLPOracle.getPrice(address token)
  • AccountManager.borrow(address account, address token, uint256 amt)
  • AccountManager.exec(address account, address target, uint256 amt, bytes data)
  • AccountManager.deposit(address account, address token, uint256 amt)

测试侧分工

  • test/helpers/SentimentAddresses.sol：保存 fork 区块、攻击地址和协议地址簿
  • test/interfaces/SentimentInterfaces.sol：只保留 PoC 需要调用的最小接口
  • test/poc.t.sol：保存 fork 初始化、攻击阶段划分和 fallback() 骨架

源码树说明

  • src/sentiment/modules/AccountManager/ 来自 0xb085...C093 对应的已验证源码
  • src/sentiment/modules/WeightedBalancerLPOracle/ 来自 0x16F3...1b5B 对应的已验证源码
  • 为了让 Foundry 能直接编译，只在根 foundry.toml 里补了 remapping，没有再在 src/ 下额外放 README
