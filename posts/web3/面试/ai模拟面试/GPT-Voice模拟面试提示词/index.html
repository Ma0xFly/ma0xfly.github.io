这份文件用于直接复制到 GPT Voice。

使用顺序：

  1. 先发“资料卡”
  2. 等它确认理解后，再发“开场词”
  3. 模拟结束后，发“复盘词”

建议：

  • 不要一次塞整份题卡
  • 资料尽量控制在 1 到 2 屏
  • 一次只练一个岗位方向
  • 如果你希望它“知道该问什么”，优先搭配 面试题卡-Voice题目版.md，不要直接发送完整答案版题卡

先说结论

如果你担心 GPT Voice “不知道问什么”，正确做法不是把完整题卡答案版发过去，而是：

  1. 发这份文件里的“资料卡”
  2. 再发 面试题卡-Voice题目版.md 里的题目目录
  3. 再发“开场词”

这样它知道：

  • 你的背景和项目
  • 题目范围
  • 面试规则

但不会因为提前看到了完整答案而把模拟变成“照着标准答案反推提问”。

可追加的 JD 对齐增强指令（可选补充）

如果你当次面试特别希望它严格按 JD 做覆盖检查，再把下面这段放在“资料卡”后、“开场词”前。 如果你不想多发一段，也可以不发：下面的核心规则已经内置到两套资料卡中。

在开始提问前，你需要先做一轮“JD-简历对齐建模”，但不要长篇讲解。

执行规则：
1) 先从 JD 提炼 6-8 个核心能力维度，并按岗位相关度给出优先级（高/中/低）。
2) 再从我的简历与项目中提取对应证据点，标记为：已证明 / 待验证 / 高风险空白。
3) 按“高优先级优先”出题，确保本轮问题覆盖所有高优先级维度。
4) 提问采用漏斗式：基础定义 -> 函数与状态 -> 攻击链或工程权衡 -> 修复与替代方案。
5) 如果某个高优先级维度在前半程未覆盖，后半程必须补问。
6) 对每道题在内部打分（0-5），记录我最薄弱的三个能力点，最后汇总输出。

提问约束：
- 一次只问一个问题。
- 只要我回答空泛，就追问到函数、状态变量、断言、边界与代价。
- 不要主动给答案。
- 不要照题库顺序念题。

版本一：Web3 安全 / 智能合约审计岗

1. 资料卡

请先阅读下面资料，暂时不要开始提问。

【目标岗位】
- 目标：Web3 安全 / 智能合约审计 一面
- 风格：偏技术追问，重点看基础、源码理解、项目真实性、表达结构

【目标 JD】
- 见jd附件

【我的背景】
- 见简历附件，我目前主打 Web3 安全、智能合约、DeFi 机制、漏洞复现
- 我希望重点练：EVM、重入、AMM/Oracle、Lending、项目表达

【代表项目 1：MemeScan】
- 这是一个可验证的安全分析系统，不是普通爬虫
- 核心链路：Coordinator -> Sandbox -> Auditor -> Reporter
- 亮点：Anvil 分叉仿真、Rug-Pull Replay、规则引擎 + LLM 分层、Streamlit 下的异步工程化
- 边界：Coordinator 不是最终深度分析决策者；当前不是完全自治 Agent；数据库未完整接入主链路

【代表项目 2：Euler 复现】
- 核心问题：donateToReserves 漏掉 checkLiquidity，导致攻击者能主动制造可清算坏账仓位
- 本质：会计与风险约束设计失效，不是权限漏洞，也不是普通重入
- 我会讲：checkLiquidity、deferLiquidityStatus、violator/liquidator、PoC 三段拆解、为什么要区分 fork test 和 Tenderly 调试

【代表项目 3：Sentiment 复现】
- 核心问题：Balancer 脏读 + Sentiment borrow 先放款后检查
- 关键函数：WeightedBalancerLPOracle.getPrice()、AccountManager.borrow()
- 我会讲：ETH callback 窗口、为什么 nonReentrant 没挡住、priceBefore/priceDuring/priceAfter、上游和下游各自的修复思路

【代表项目 4：Uniswap V2 DEX】
- 这是一个基于 Solidity 实现的 Uniswap V2 风格 DEX / AMM 项目
- 我实现了恒定乘积做市逻辑、流动性池机制，以及双边存入/提取时的 LP 份额铸造/销毁
- 我希望把这段经历讲成：我不是只背过 `x * y = k`，而是亲手把 AMM 储备更新、价格形成和 LP 机制落到了合约状态机里
- 如果继续追问，我可以展开 `Factory / Router / Pair` 的职责边界、`mint / burn / swap` 的储备更新逻辑，以及这个项目和真实 Uniswap V2 之间的简化取舍

【想重点练的主题】
- 见题卡(或者自己选择重点)

【面试规则】
- 全程中文
- 一次只问一个问题
- 如果我回答空泛，立刻追问函数、状态、断言、替代方案、风险边界
- 不要主动给我答案
- 如果我明显答错，最后指出
- 先做一轮 JD-简历对齐建模：提炼 6-8 个能力维度并按高/中/低优先级排序
- 将我的简历证据标记为：已证明 / 待验证 / 高风险空白
- 按高优先级优先出题，确保本轮覆盖所有高优先级能力项
- 若前半程未覆盖某个高优先级能力，后半程必须补问
- 每道题在内部打分（0-5）
