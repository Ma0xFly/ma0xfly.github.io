预言机操纵漏洞 (Oracle Manipulation)

标签: #漏洞类型/价格操纵 | #协议类型/DeFi/借贷/合成资产 | #严重程度/Critical

📌 1. TL;DR (一句话快速回顾)

  • 本质成因：智能合约无法直接感知外部世界的资产价格，当协议单一、直接、实时地依赖某个流动性不深的链上现货池（如 Uniswap V2 瞬时价格）作为核心资产计价依据时，该价格就极易被扭曲。
  • 攻击路径：攻击者利用自有巨资或闪电贷，在目标预言机池中进行天量单向交易（砸盘或拉盘），人为制造出一个极端脱离公允价值的“假报价”，趁机在受害者协议中“高价抵押”或“低成本清算”。

🔍 2. 漏洞底层原理 (Root Cause)

正常业务逻辑

  • 借贷协议（如 Compound）为了判断用户的抵押物（比如 100个 ETH）能借出多少稳定币，需要去问预言机：“现在 1 个 ETH 值多少 DAI？” 预言机返回价格，协议据此放款。

缺陷剖析

  • 致命代码/机制：开发者图省事，直接写了一个 getPrice() 函数去读取 Uniswap 池子里的 balance(DAI) / balance(ETH) 当作报价。
  • 为何产生冲突：
    • 预期状态：Uniswap 池子的比例代表了市场的公允价格。
    • 实际状态：攻击者在这个区块内，刚刚往该池子里砸了 10 万个 ETH。此时池子里的 ETH 泛滥成灾，DAI 被抽干。Uniswap 给出的瞬时价格变成了 1 ETH = 0.001 DAI。受害借贷协议如果在这个时候去读价格，就会认为 ETH 暴跌了。

⚔️ 3. 攻击执行路径 (Exploit Steps)

最典型的玩法：闪电贷 + 瞬时预言机操纵

  1. [准备阶段]: 攻击者通过闪电贷借出 1 亿枚 USDC 和 1 万枚 TokenA。
  2. [触发阶段 (制造价格极值)]: 攻击者去 Uniswap V2 的 TokenA/USDC 交易对，把 1 亿枚 USDC 全部砸进去买入 TokenA。这导致该池子中 TokenA 的瞬时价值被拉升了无数倍（比如原本 1 A = 1 USDC，现在被拉到了 1 A = 10000 USDC）。
  3. [利用阶段 (薅羊毛)]: 攻击者拿着本来不值钱的 TokenA，存入借贷协议。借贷协议去读 Uniswap 的现价：“哇，1 个 TokenA 值 10000 USDC！” 于是，借贷协议认为攻击者是大富翁，允许他把协议国库里的 WETH、WBTC 全部借空。
  4. [获利退场]: 攻击者把刚买的 TokenA 在 Uniswap 砸回去换回 USDC（或者直接不换了亏掉这点手续费），归还闪电贷。剩下的 WBTC、WETH 成为纯利润。借贷协议剩下了一堆高估值的垃圾 TokenA 坏账。

💻 4. 关键代码示例 (Snippets)

漏洞代码 (Vulnerable Code)

// 危险！这就是一个最脆弱的现价预言机实现
function getUnderlyingPrice(address cToken) external view returns (uint) {
    address pair = IUniswapV2Factory(factory).getPair(cToken, WETH);
    
    // 直接拿余额相除！随时可以被单笔交易的闪电贷拉爆
    uint reserve0 = IERC20(cToken).balanceOf(pair);
    uint reserve1 = IERC20(WETH).balanceOf(pair);
    
    return reserve1 * (10 ** 18) / reserve0; 
}

🛡️ 5. 修复与防范方案 (Mitigations)

  • 协议自身的修复：
    • (首选) 使用去中心化预言机网络：如 Chainlink 等节点报价网络。Chainlink 的价格是无数个节点在链下抓取各大中心化交易所（Binance, Coinbase等）的均价提交上来的，你很难用链上的一笔闪电贷去影响它。
    • 不要使用瞬时现价 (Spot Price)：如果你非要用 AMM 来报价，必须使用时间加权平均价（TWAP）。（但注意：TWAP 也有它的克星，详见下一篇笔记）。
    • 引入多源价格校验机制：同时抓取 Chainlink、Uniswap TWAP、Curve 的价格。当几个源的价格偏差大于某个阈值（例如 5%）时，立刻熔断协议暂停借贷清算。

📖 6. 真实案例与靶场 (References)

  • 经典黑客事件：Mango Markets 被黑事件（1.1亿美金）, DeFi 发展史上最泛滥的死法。早期诸如 bZx, Harvest Finance, Value DeFi 等被反复收割。
  • 靶场/练习：Damn Vulnerable DeFi - [Puppet], [Puppet v2] (这俩生动演示了分别用 Uniswap V1 和 V2 搞预言机操纵的惨案)。
  • 拓展阅读：预言机安全不仅是代码问题，更是经济学问题。当操纵预言机的成本（滑点、手续费）低于你在受害者协议里薅出来的利润时，攻击必然发生。
