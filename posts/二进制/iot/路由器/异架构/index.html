mips架构

java -jar Mars4_5.jar：打开mars

mips特点

流水线效应：MIPS采用了高度的流水线，其中最重要的就是分支延迟效应。在分支跳转语句后面那条语句叫分支延迟槽。实际上，在程序执行到分支语句时，当他刚把要跳转的地址填充好（填充到代码计数器里），还没有完成本条指令时，分支语句后面的那个指令就已经执行了，其原因就是流水线效应——几条指令同时执行，只是处于不同的阶段，mips不像其它架构那样存在流水线阻塞。所以分支跳转语句的下一条指令通常都是空指令nop或一些其他有用的语句。

缓存刷新机制：MIPS CPUs有两个独立的cache:指令cache和数据cache。 指令和数据分别在两个不同的缓存中。当缓存满了，会触发flush, 将数据写回到主内存。攻击者的攻击payload通常会被应用当做数据来处理，存储在数据缓存中。当payload触发漏洞， 劫持程序执行流程的时候，会去执行内存中的shellcode.如果数据缓存没有触发flush的话，shellcode依然存储在缓存中，而没有写入主内存。这会导致程序执行了本该存储shellcode的地址处随机的代码，导致不可预知的后果。(通常执行sleep(1)刷新)

mips架构的题目有一个特性，没有办法开始NX保护，那么我们除了常规构造ROP链子，还可以通过写shellcode。

  1. 纯ROP链，通过调用system函数来getshell 。

  2. 通过构造ROP链，跳转至读入到栈/bss段等处的shellcode执行。

函数调用规则

前四个参数：

  • 前四个参数通过寄存器 $a0、$a1、$a2 和 $a3 传递。

  • 如果函数的参数少于四个，多余的寄存器不会被使用。

超过四个参数：

  • 如果函数的参数超过四个，额外的参数会通过栈传递。

  • 调用者需要在栈上为这些额外的参数分配空间，并将参数值存储到栈上。

寄存器：

前4个参数：a0,a1,a2，a4

返回地址：$ra

pc : rip

$zero：总是0，方便进行各种运算

rax: v0 system的系统调用是0xfab

$sp：rsp

$fp: rbp

$at：用于标记当前的汇编指令。

v0−v1：用于存储函数的返回值。

t0−t9：通用寄存器。

s0−s7：用于保存局部变量的寄存器。

$t8：用于保存程序计数器（PC）

汇编指令集：

跳转类

jr +寄存器:跳转到寄存器(Jump Register)

jal displayMessage ;跳转指令调用并返回，1、跳转displayMessage地址，同时把下一条指令的地址写入到 $ra寄存器

jalr+寄存器：jalr $t9 # 等价于 jalr $ra, $t9，跳转到寄存器中存的地址，同时把当前 PC + 8(32位是4，即延迟槽后）保存到 $ra，注意：MIPS有延迟槽，执行完 jalr 后，下一个指令（1条）仍会执行，然后才跳转。

li $t9, 0x00401234
jalr $t9
nop            # 延迟槽：这里的指令会在跳转前执行

执行完后：
跳转到 0x00401234
$ra = 当前地址 + 8

加载类

li：类lea但是是加载立即数的

la: 类似lea载入地址

sw: 将寄存器（s0）中的值放入$sp + 4 * 倍 数位置 sw $s0,0($sp）这里sp前的0表示0倍

sw $rt, offset($rs)
  • $rt：源寄存器，其中存储着要写入内存的数据。
  • offset：16 位有符号立即数，用于表示偏移量。
  • $rs：基址寄存器，提供基地址。
  • 有效地址计算：内存地址 = $rs + offset
# 函数入口：保存寄存器到栈
addi $sp, $sp, -12     # 为3个寄存器分配空间（每个4字节）
sw $ra, 8($sp)         # 保存返回地址（$ra）
sw $s0, 4($sp)         # 保存$s0（被调用者保存寄存器）
sw $s1, 0($sp)         # 保存$s1

# 函数体...

# 函数出口：恢复寄存器
lw $s1, 0($sp)         # 恢复$s1
lw $s0, 4($sp)         # 恢复$s0
lw $ra, 8($sp)         # 恢复返回地址
addi $sp, $sp, 12      # 释放栈空间
jr $ra                 # 返回

lw:从栈上取回值到寄存器 lw $s0,0($sp),从离sp指针0偏移的地址值给s0

计算类

addi $s0,$zero,5：将加法结果加到第一个寄存器

add $a0,$zero,$s0：将加法结果赋值给第一个寄存器

addiu rd, rs, immediate: 将寄存器 rs 中的值与一个立即数（immediate）相加，并将结果存入目标寄存器 rd 中。

arm架构

ARM 特点

流水线效应：ARM 采用多级流水线设计（如三级流水线：取指、译码、执行，现代架构可达 10 级以上）。与 MIPS 的分支延迟槽不同，ARM 通过分支预测技术减少流水线阻塞，多数情况下无需处理延迟槽，仅在早期架构或特定模式下存在类似机制。

缓存刷新机制：ARM 处理器包含指令缓存（I-Cache）和数据缓存（D-Cache），部分架构支持统一缓存。缓存管理通过专用指令（如dcbst清理数据缓存、icbi无效化指令缓存）进行控制。在漏洞利用中，若 shellcode 存于数据缓存未同步到主存，会导致执行失败，需通过dsb（数据同步屏障）、isb（指令同步屏障）确保数据可见性。

ARM具备独立的指令Cache与数据Cache。在exploit过程中，如果payload以数据方式写入cache但未刷新至主内存，那么代码执行会失败。对策类似：在写入payload后添加如sleep()等系统调用使数据flush到主存，确保shellcode生效。

安全特性：现代 ARM 架构支持 NX（No-eXecute）保护、ASLR（地址空间布局随机化）、TrustZone 技术等。这使得漏洞利用更依赖 ROP 技术，而直接执行 shellcode 需绕过 NX 保护。

  1. 纯ROP链，通过调用system函数来getshell。
  2. 利用内存映射机制，将数据区域标记为可执行，再执行 shellcode。

函数调用规则

前四个参数：

  • 前四个参数通过寄存器r0、r1、r2和r3（32 位）或x0、x1、x2、x3（64 位）传递。
  • 如果函数的参数少于四个，多余的寄存器不会被使用。

超过四个参数：

  • 如果函数的参数超过四个，额外的参数通过栈传递从右至左依次入栈
  • 调用者需要在栈上为这些额外的参数分配空间，并将参数值存储到栈上，且栈需保持 8 字节或 16 字节对齐。

寄存器：

前 4 个参数：r0、r1、r2、r3（32 位）；x0、x1、x2、x3（64 位） 返回地址：r14（LR，32 位）；x30（LR，64 位） pc：程序计数器，指向当前执行指令 r0：用于存储函数的返回值（32 位）；x0 用于存储函数的返回值（64 位） r1-r3：除传递参数外，也可作为通用寄存器（32 位）；x1-x3 同理（64 位） r4-r11：被调用者保存寄存器，用于保存局部变量（32 位）；x4-x15 同理（64 位）

r11(fp): 栈底指针

r12（IP）：临时寄存器，用于中间计算（32 位）；x16-x17（IP0-IP1）同理（64 位） r13（SP）：栈指针，指向栈顶（32 位）；x18（SP）同理（64 位） r14（LR）：链接寄存器，保存函数返回地址（32 位）；x30（LR）同理（64 位） r15（PC）：程序计数器，指向当前执行指令（32 位）；x31（PC）同理（64 位） CPSR：当前程序状态寄存器，存储标志位（如零标志 Z、进位标志 C 等，32 位）；PSTATE 同理（64 位）

汇编指令集：

跳转类

b + 标签：无条件跳转到标签处（Branch） bl 标签：跳转并链接，跳转至标签地址，同时把下一条指令的地址写入到 LR 寄存器（r14） bx + 寄存器：跳转到寄存器中的地址，并根据寄存器最低位切换指令集（0 为 ARM，1 为 Thumb） blx + 寄存器：跳转到寄存器中的地址，同时把下一条指令的地址写入到 LR 寄存器，并切换指令集

ldr r0, =0x00401234
blx r0
; 执行完后：
; 跳转到0x00401234
; LR = 当前地址 + 4

加载类

ldr 寄存器，= 立即数：加载立即数到寄存器（类似 MIPS 的 li） ldr 寄存器，地址：加载地址中的值到寄存器（类似 MIPS 的 la） str 源寄存器，[基址寄存器，偏移量]：将源寄存器中的值存储到基址寄存器加偏移量的内存位置

ldr r0, =0x12345678     ; 加载立即数地址（类似li）
ldr r0, [r1, #4]        ;将地址r1 + 4处的数据加载到寄存器r0中。
ldr r1, [r2]            ; 加载内存中的值到r1
str r3, [sp, #8]        ; 把r3的值存储到 sp+8 的地址

计算类

add 目标寄存器，源寄存器 1, 源寄存器 2：将源寄存器 1 和源寄存器 2 的值相加，结果存入目标寄存器 add 目标寄存器，源寄存器，立即数：将源寄存器的值与立即数相加，结果存入目标寄存器 sub 目标寄存器，源寄存器 1, 源寄存器 2：将源寄存器 1 的值减去源寄存器 2 的值，结果存入目标寄存器 sub 目标寄存器，源寄存器，立即数：将源寄存器的值减去立即数，结果存入目标寄存器

add r0, r1, r2      ; r0 = r1 + r2
add r0, r0, #5      ; r0 = r0 + 5
sub sp, sp, #16     ; sp = sp - 16，为栈分配16字节空间

栈操作类

  • push：将一个或多个寄存器的值压入栈中。

    push {r0, r1, lr}
    • 将寄存器r0、r1和lr的值压入栈中。

  • pop：从栈中弹出一个或多个寄存器的值。

    pop {r0, r1, pc}
    • 从栈中弹出值到寄存器r0、r1和pc中。如果弹出到pc，程序将跳转到弹出的地址继续执行。
; 函数入口：保存寄存器到栈
push {r4, r5, lr}       ; 保存r4、r5和lr到栈中
; 函数体...
; 函数出口：恢复寄存器
pop {r4, r5, pc}        ; 恢复r4、r5，并将返回地址弹出到pc，返回
