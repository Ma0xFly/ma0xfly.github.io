ADB

1.环境&连接

| 查看ADB版本 | adb version | 确认本地环境正常 |
| 列出已连接设备 | adb devices | 结果含 `device` 表示已授权 |
| 重启ADB服务 | adb kill-server  adb start-server | 解决端口占用或识别异常 |
| 连接无线设备 | adb connect IP:PORT | 默认端口 5555 |
| 断开无线设备 | adb disconnect IP:PORT | 省略 IP 断开全部 |

2. 文件传输

貌似不能在adb shell下传输

| 推送文件到设备 | adb push 本地路径 设备路径
| 拉取文件到电脑 | adb pull 设备路径 本地路径
adb push app-debug.apk /sdcard/
adb pull /sdcard/log.txt ./log.txt

3. Shell & 权限

目的 指令
进入交互 Shell adb shell
执行单条命令 adb shell <command>
获取 root 权限 adb root
重新挂载 /system 可读写 adb remount
退出 Shell exit

4. 应用管理

目的 指令
安装 APK adb install [-r] [-d] <file.apk>
卸载应用 adb uninstall [-k] <package>
列出已装包名 adb shell pm list packages
清除应用数据 adb shell pm clear <package>

5. 日志与调试

目的 指令
实时查看日志 adb logcat
仅查看崩溃日志 adb logcat -b crash
导出日志到文件 adb logcat -d > log.txt
查看内核日志 adb shell dmesg

6. 系统信息

目的 指令
查看系统版本 adb shell getprop ro.build.version.release
查看厂商型号 adb shell getprop ro.product.model
查看序列号 adb get-serialno
查看CPU架构 adb shell getprop ro.product.cpu.abi

7. 输入事件（自动化测试）

目的 指令
点击屏幕 adb shell input tap x y
滑动屏幕 adb shell input swipe x1 y1 x2 y2 [ms]
输入文本 adb shell input text "hello"
按键事件 adb shell input keyevent KEYCODE_HOME

8. 网络

目的 指令
查看IP adb shell ifconfig 或 ip addr show wlan0
端口转发 adb forward tcp:本地端口 tcp:设备端口
反向转发 adb reverse tcp:设备端口 tcp:本地端口

9. 备份 & 恢复

目的 指令
创建完整备份 adb backup -apk -shared -all -f backup.ab
恢复备份 adb restore backup.ab

10. 快捷别名（可选）

在 ~/.bashrc 或 ~/.zshrc 中添加：

alias adbd="adb devices"
alias adblog="adb logcat -v color"
alias adbshell="adb shell"
